# 投产材料管理系统 \- 核心业务逻辑脱敏原型

该项目是对本人实习期间，负责开发的“投产材料管理”模块的后端功能复现与个人总结。
在开发过程中，保留了模块核心业务逻辑（含递归机构树、旁路缓存、AOP审计、流式导出等关键功能），
同时对所有银行敏感信息、企业级专属配置及私有API进行了深度脱敏处理，仅用于个人技术积累与展示。

## 核心技术栈

- 后端框架：Spring Boot 3.2.4 Maven 3.8.1

- 持久层：MyBatis-Plus 3.5.5

- 中间件：Redis 8.4.0

- 功能组件：EasyExcel（处理数据流式导出）、Spring AOP（实现非侵入式审计日志）

- 工具类：自定义ZipUtils（多线程文件压缩）、Fastjson2（JSON序列化与反序列化）

- 测试支撑：JUnit 5 + Mockito

## 技术亮点与实际解决方案

### 1\. 高性能组织架构检索（递归算法 \+ Redis 旁路缓存）

针对银行“总行\-分行\-支行”多级机构的复杂层级关系，以及跨机构数据检索效率低的问题，我设计了O(1)复杂度的检索方案，具体实现如下：

- 递归逻辑：自定义fillChildIds递归算法，可穿透机构无限层级，高效解决跨机构权限隔离场景下的数据归集问题，确保数据检索的完整性。

- 缓存策略：引入Cache Aside（旁路缓存）机制，优化查询性能。查询时优先读取Redis缓存，缓存未命中则递归查询数据库，并延迟重建缓存；
          当机构数据发生变更时，通过clearDeptCache方法主动失效对应缓存，确保缓存与数据库数据强一致。

- 性能提升：通过缓存优化，将原本依赖数据库的深层级联计算，转为内存直接查询，大幅提升系统体验。

### 2\. 非侵入式自动化审计日志（Spring AOP）

结合银行系统对关键操作可追溯的核心需求（如PRD修改、附件删除等敏感操作），我采用Spring AOP实现了非侵入式审计日志功能：

- 自定义注解：设计@Log业务注解，通过AspectJ切面拦截标注该注解的目标方法，无需侵入业务代码即可捕获操作日志。

- 异步持久化：捕获的操作日志通过异步线程存入sys\_oper\_log表，避免阻塞主业务流程，确保关键操作100%留痕，满足银行合规要求。

### 3\. 文件处理流水线与OOM风险规避

针对投产材料准备阶段，大批量附件管理、大报表导出易出现内存溢出的问题，我设计了高效的文件处理方案：

- 流式导出：放弃原生POI，集成EasyExcel实现逐行读取、逐行导出，有效规避了万级以上数据导出时可能出现的JVM内存溢出（OOM）风险，保障系统稳定运行。

- 多维附件管理：实现附件批量上传、物理文件与业务数据的关联绑定，同时开发多线程Zip压缩下载功能，提升附件管理效率。

## 项目目录结构说明

```text
src/main/java/com/example/prd
├── annotation      # 自定义业务注解（如@Log审计注解）
├── aspect          # AOP切面实现（审计日志捕获、切面逻辑）
├── common          # 通用工具封装（统一响应Result、异常处理等）
├── config          # 全局配置（Redis缓存配置、文件上传路径配置等）
├── controller      # 业务接口层（PRD清单、部门管理、日志查询等接口）
├── entity          # 数据库实体类（与数据表一一映射）
├── mapper          # MyBatis接口及SQL映射文件（数据库操作层）
├── service         # 核心业务逻辑层（递归算法、缓存控制、附件处理等）
└── utils           # 通用工具类（文件压缩、Excel处理、日期工具等）

src/main/resources
├── sql             # 数据库初始化脚本（提供完整 DDL）
│   ├── prd_check_list.sql  # 投产清单业务表
│   ├── sys_dept.sql        # 银行多级机构部门表
│   └── sys_oper_log.sql    # 操作日志审计表
└── application.yml # 环境配置文件

src/test/java       # 单元测试模块
└── com.example.prd
    └──  controller  # 接口 Mock 测试
```

## 快速启动

1. 环境要求：JDK 21、MySQL 8.0+、Redis 8.x+（需提前部署对应环境）。

2. 配置调整：修改application\.yml文件中的数据源、Redis连接信息。

3. 项目运行：直接执行BankwebApplication启动类，启动成功后即可访问相关业务接口。
